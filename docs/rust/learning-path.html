🦀 Rust 学习路线（从入门到能写生产级服务）

这条路线适合已经精通 Java 并发、Spring、Netty 的后端工程师，旨在通过"结构化、无废话、可实操"的风格，帮助你顺滑地过渡到 Rust 开发。

阶段 0：环境准备（1 天）​

目标：能跑代码、能查文档、能管理依赖

📚 推荐阅读：环境搭建 | Cargo 指南

  • 安装 rustup（版本管理）
  • 安装 cargo（构建 + 包管理）
  • 配置 VSCode + rust-analyzer
  • 学会 cargo new / run / test / build / fmt / clippy

阶段 1：Rust 核心语法（3–5 天）​

目标：能写小程序，理解 Rust 的基本语法

📚 推荐阅读：基础语法 | 结构体和枚举 | 集合 | 错误处理 | 模块系统

重点内容：

  • 基础语法：变量、模式匹配、表达式、枚举、结构体
  • 所有权（Ownership）
  • 借用（Borrowing）
  • 生命周期（Lifetime）
  • 模块系统（mod / use / crate）
  • Result / Option 错误处理
  • 常用集合：Vec、HashMap、String

练习建议： 写一个 CLI 小工具（如：文件搜索器、日志过滤器）

阶段 2：所有权 + 生命周期（核心难点）（7–10 天）​

目标：真正理解 Rust 的灵魂

📚 推荐阅读：所有权系统 | 生命周期 | 闭包和迭代器

重点内容：

  • Move / Copy / Clone
  • 借用规则：可变借用 vs 不可变借用
  • 生命周期省略规则
  • 引用悬垂的编译器检查
  • Slice、Iterator、闭包捕获规则

练习：

  • 手写一个简单的 LRU Cache
  • 手写一个 mini HTTP parser（只解析请求行）

阶段 3：并发与多线程（5–7 天）​

目标：掌握 Rust 的线程安全模型

📚 推荐阅读：并发编程 | 智能指针

重点内容：

  • Send / Sync trait
  • Arc / Mutex / RwLock
  • Atomic 原子类型
  • Channel（std + crossbeam）
  • 零成本抽象下的并发性能

练习：

  • 写一个并发计数器
  • 写一个多线程日志聚合器
  • 写一个线程池（Tokio 前的铺垫）

阶段 4：Rust 异步体系（10–14 天）​

目标：能写高性能异步服务

📚 推荐阅读：异步编程

重点内容：

  • Future / Poll / Waker 原理
  • Pin / Unpin
  • async/await 的编译器展开
  • Tokio（最主流 runtime）
  • 异步 IO、定时器、任务调度
  • 异步 Mutex / RwLock

练习：

  • 用 Tokio 写一个 mini HTTP server
  • 写一个异步爬虫
  • 写一个异步任务调度器

阶段 5：工程化能力（7–10 天）​

目标：能写生产级 Rust 项目

📚 推荐阅读：工程化实践 | 项目组织 | 测试 | 最佳实践

重点内容：

  • cargo workspace
  • feature flag
  • error handling（thiserror / anyhow）
  • logging（tracing）
  • 配置管理（config）
  • 序列化（serde）
  • 单元测试 / 集成测试
  • 文档注释（rustdoc）

练习：

  • 写一个 CLI + HTTP 的混合项目
  • 写一个可配置的服务框架（类似 mini-Spring Boot）

阶段 6：Web 后端开发（10–20 天）​

目标：能写真正的后端服务

📚 推荐阅读：Web 后端开发 | 实战项目

推荐框架：

  • Axum（最现代、生态最好）
  • Actix-web（性能极高）

学习内容：

  • 路由、提取器、中间件
  • Tower 服务栈
  • 数据库（SQLx / SeaORM）
  • Redis（deadpool-redis）
  • JWT / OAuth
  • 高并发优化

练习：

  • 写一个完整的用户系统（登录、权限、会话）
  • 写一个高并发短链服务
  • 写一个 WebSocket 实时服务

阶段 7：深入底层（可选，10–20 天）​

📚 推荐阅读：底层原理 | Unsafe Rust | 智能指针 | 宏编程

适合喜欢研究原理的开发者：

  • Unsafe Rust
  • 内存布局
  • 自定义智能指针
  • 自己实现 Future
  • 自己实现一个 mini runtime
  • FFI（与 C/Java 交互）

🧭 总结：Rust 学习路线图​

阶段 0：环境准备
阶段 1：基础语法
阶段 2：所有权 + 生命周期（核心）
阶段 3：并发（Send/Sync）
阶段 4：异步（Future/Tokio）
阶段 5：工程化（workspace、error、logging）
阶段 6：Web 后端（Axum）
阶段 7：底层（Unsafe、FFI）
