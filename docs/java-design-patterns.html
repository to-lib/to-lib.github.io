Java 设计模式完全指南

适用版本: JDK 1.8 (Java 8) 及以上

本文档所有代码示例基于 JDK 1.8 编写，充分利用了 Java 8 的新特性（Lambda 表达式、Stream API、函数式接口等），同时确保向后兼容更高版本的 JDK。

环境要求
  • JDK 版本: JDK 1.8 或更高版本
  • 构建工具: Maven 3.6+ 或 Gradle 6.0+
  • IDE: IntelliJ IDEA / Eclipse / VS Code

👉 详细配置请参考 环境配置指南

欢迎来到 Java 设计模式完全学习指南！本文档包含了 23 种经典设计模式的详细讲解、实际应用示例和最佳实践。

📚 文档组织结构​

本指南分为以下几个部分：

📖 基础入门​

  • 设计模式概述 - 理解什么是设计模式及其分类
  • 快速参考 - 23 种模式的快速查询表

🏭 创建型模式 (5 种)​

构造对象，隐藏创建逻辑

  1. 单例模式 (Singleton) - 确保类仅有一个实例
  2. 工厂方法 (Factory Method) - 创建对象的接口
  3. 抽象工厂 (Abstract Factory) - 创建产品族
  4. 建造者模式 (Builder) - 分步构建复杂对象
  5. 原型模式 (Prototype) - 通过克隆创建对象

🔧 结构型模式 (7 种)​

组合对象，形成更大的结构

  1. 适配器模式 (Adapter) - 转换接口实现兼容
  2. 装饰器模式 (Decorator) - 动态添加功能
  3. 外观模式 (Facade) - 简化复杂子系统
  4. 代理模式 (Proxy) - 控制对象访问
  5. 组合模式 (Composite) - 树形结构处理
  6. 享元模式 (Flyweight) - 共享细粒度对象
  7. 桥接模式 (Bridge) - 分离抽象和实现

🎯 行为型模式 (11 种)​

处理对象间的通信和职责分配

  1. 观察者模式 (Observer) - 一对多通知
  2. 策略模式 (Strategy) - 可互换的算法
  3. 状态模式 (State) - 状态改变行为
  4. 命令模式 (Command) - 将请求对象化
  5. 模板方法 (Template Method) - 定义算法骨架
  6. 责任链 (Chain of Responsibility) - 请求沿链传递
  7. 迭代器 (Iterator) - 遍历集合元素
  8. 中介者 (Mediator) - 集中管理对象通信
  9. 备忘录 (Memento) - 保存/恢复状态
  10. 访问者 (Visitor) - 为对象添加操作
  11. 解释器 (Interpreter) - 解析和执行语言

📊 设计模式分类速览​

按使用频率​

高频 ⭐⭐⭐

  • Singleton（无处不在）
  • Factory Method（对象创建）
  • Strategy（算法选择）
  • Observer（事件系统）
  • Decorator（功能增强）

中频 ⭐⭐

  • Abstract Factory
  • Builder
  • Template Method
  • Command
  • State

低频 ⭐

  • 其他 11 种模式（特定场景）

按复杂度​

  • 简单 - 容易理解：Singleton, Factory Method, Strategy, Adapter, Observer
  • 中等 - 需要经验：Abstract Factory, Builder, Decorator, Proxy, State, Command, Template Method
  • 复杂 - 需要深入理解：Bridge, Composite, Facade, Flyweight, Chain of Responsibility, Mediator, Memento, Iterator, Visitor, Interpreter, Prototype

🔍 快速查询​

我需要创建对象？

  • 单一对象 → Singleton
  • 多种类型 → Factory Method
  • 产品族 → Abstract Factory
  • 复杂对象 → Builder
  • 克隆对象 → Prototype

我需要组织对象结构？

  • 树形结构 → Composite
  • 动态功能 → Decorator
  • 简化系统 → Facade
  • 隐藏实现 → Proxy
  • 转换接口 → Adapter
  • 分享对象 → Flyweight

我需要处理对象交互？

  • 一对多通知 → Observer
  • 可换算法 → Strategy
  • 状态转换 → State
  • 多对多通信 → Mediator
  • 其他...
