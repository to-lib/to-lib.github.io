PostgreSQL 简介

PostgreSQL 是一个功能强大的开源对象关系型数据库系统，具有超过 35 年的积极开发历史，在可靠性、功能健壮性和性能方面赢得了良好的声誉。

🎯 学习目标​

本教程将帮助你：

  • 掌握 PostgreSQL 的核心概念和基础知识
  • 学习 SQL 语法和数据类型
  • 理解索引、事务和并发控制
  • 掌握性能优化和备份恢复
  • 了解高级特性和最佳实践

✨ 主要特性​

1. 完全符合 ACID​

  • 原子性（Atomicity）：事务要么完全执行，要么完全不执行
  • 一致性（Consistency）：事务执行前后数据保持一致
  • 隔离性（Isolation）：并发事务之间相互隔离
  • 持久性（Durability）：已提交的事务永久保存

2. 丰富的数据类型​

  • 基本类型：整数、浮点数、字符串、布尔值、日期时间
  • 高级类型：数组、JSON/JSONB、UUID、范围类型
  • 几何类型：点、线、圆、多边形等
  • 自定义类型：可创建自己的数据类型

3. 强大的扩展性​

  • 外键约束：维护数据完整性
  • 触发器和存储过程：自定义业务逻辑
  • 视图和物化视图：简化复杂查询
  • 全文搜索：内置强大的搜索功能
  • 扩展插件：PostGIS、pg_stat_statements 等

4. 高级查询功能​

  • 复杂 JOIN 操作
  • 窗口函数
  • 公用表表达式（CTE）
  • 递归查询
  • JSON 查询和操作

🚀 为什么选择 PostgreSQL？​

vs MySQL​

特性PostgreSQLMySQL
ACID 支持完全支持部分支持（取决于存储引擎）
并发控制MVCC锁机制
复杂查询优秀良好
数据类型非常丰富基本
JSON 支持JSONB（二进制存储）JSON（文本存储）
全文搜索内置基础
扩展性强大一般

使用场景​

PostgreSQL 更适合：

  • 复杂的数据分析和报表
  • 需要严格 ACID 保证的应用
  • 地理信息系统（GIS）
  • 科学计算和数据科学
  • 需要 JSON 文档存储的应用

MySQL 更适合：

  • 简单的读写密集型应用
  • Web 应用的快速开发
  • 对事务要求不高的场景

📚 学习路径​

初学者​

  1. 基础概念 - 了解数据库、表、字段等基本概念
  2. 数据类型 - 掌握 PostgreSQL 的数据类型
  3. SQL 语法 - 学习基本的 SQL 查询语句

进阶​

  1. 索引优化 - 理解索引原理和使用
  2. 事务管理 - 掌握事务和并发控制
  3. 性能优化 - 提升数据库性能

高级​

  1. 备份恢复 - 学习数据备份和恢复策略
  2. 复制与高可用 - 配置主从复制和高可用
  3. 安全管理 - 用户权限和安全最佳实践

🛠️ 快速开始​

安装 PostgreSQL​

macOS (使用 Homebrew):

# 安装
brew install postgresql@16

# 启动服务
brew services start postgresql@16

# 检查状态
psql --version

Ubuntu/Debian:

# 添加仓库
sudo sh -c 'echo "deb http://apt.postgresql.org/pub/repos/apt $(lsb_release -cs)-pgdg main" > /etc/apt/sources.list.d/pgdg.list'
wget --quiet -O - https://www.postgresql.org/media/keys/ACCC4CF8.asc | sudo apt-key add -

# 安装
sudo apt-get update
sudo apt-get install postgresql-16

# 启动服务
sudo systemctl start postgresql

Docker:

# 运行 PostgreSQL 容器
docker run --name postgres \
  -e POSTGRES_PASSWORD=mysecretpassword \
  -p 5432:5432 \
  -d postgres:16

# 连接数据库
docker exec -it postgres psql -U postgres

第一个查询​

-- 创建数据库
CREATE DATABASE myapp;

-- 连接到数据库
\c myapp

-- 创建表
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 插入数据
INSERT INTO users (username, email)
VALUES ('john_doe', 'john@example.com');

-- 查询数据
SELECT * FROM users;
